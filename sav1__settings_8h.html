SAV1_API void sav1_default_settings ( Sav1Settings *  settings,
char *  file_path 
)

Populate settings struct with defaults.

This is the easiest way to get your settings up and running to create a Sav1Context with. If you don't call sav1_default_settings, you have to set every setting youself. It's recommended to use sav1_default_settings to populate your settings struct, and then changing any settings you desire from the defaults after.

  • Sav1Settings::codec_target defaults to SAV1_CODEC_AV1 | SAV1_CODEC_OPUS
  • Sav1Settings::desired_pixel_format defaults to SAV1_PIXEL_FORMAT_RGBA
  • Sav1Settings::queue_size defaults to 20
  • Sav1Settings::frequency defaults to SAV1_AUDIO_FREQ_48KHZ
  • Sav1Settings::channels defaults to SAV1_AUDIO_STEREO
  • Sav1Settings::playback_mode defaults to SAV1_PLAYBACK_TIMED
  • Sav1Settings::on_file_end defaults to SAV1_FILE_END_WAIT
  • Sav1Settings::playback_speed defaults to 1.0
Parameters
[in]settingspointer to a SAV1 settings struct that will be modified
[in]file_pathpath to the file that will be played

Set up custom audio processing for every Sav1AudioFrame.

Set up a custom audio processing function to be applied to every Sav1AudioFrame in an additional thread before it reaches the user. This can be done manually by modifying the Sav1Settings struct directly, but this function provides a convenient way to do so. New frames are not created, but the existing frame passed in can be modifed and the Sav1AudioFrame::custom_data property can be used to store additional output data. Include an optional cookie to store necessary information.

If destroy_function is NULL, then no additional steps will be taken when frames are destroyed. If memory is allocated for the Sav1AudioFrame::custom_data property, then a function should be passed for this argument that will free that memory.

If cookie is NULL, then NULL will be passed in for the cookie argument of the processing_function and the destroy_function. The cookie can be used to store additional context that is necessary for applying whatever postprocessing is desired.

Parameters
[in]settingspointer to a SAV1 settings struct
[in]processing_functionfunction to modify Sav1AudioFrames and add custom data
[in]destroy_functionfunction for destroying custom Sav1AudioFrame data or NULL
[in]cookiedata to be passed in during custom frame processing and destruction or NULL
